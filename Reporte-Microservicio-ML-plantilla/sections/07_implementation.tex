\subsection{Arranque del servicio}

Para levantar todo esto en el servidor (o en local), los comandos están súper sencillos porque usamos \texttt{venv} y FastAPI:

\begin{lstlisting}[style=bashstyle]
# Crear el entorno y activar
python3 -m venv venv
source venv/bin/activate

# Instalar dependencias
pip install -r requirements.txt
python -m spacy download es_core_news_sm

# Levantar el server en el puerto 3000
uvicorn app.api.server:app --host 0.0.0.0 --port 3000 --reload
\end{lstlisting}

\subsection{Manejo de errores}

\begin{itemize}
  \item \texttt{400 Bad Request}: Si el documento viene incompleto (faltan los objetivos o el alcance) o si es un PDF escaneado del que no pudimos sacar texto.
  \item \texttt{422 Unprocessable Entity}: Si el frontend trata de pegarle al endpoint pero no manda el archivo (FastAPI rebota esto en automático).
  \item \texttt{500 Internal Server Error}: Si se crashea el LLM o falla Ollama por RAM, lo atajamos y devolvemos un mensaje claro para que la app móvil no se quede pasmada.
\end{itemize}

\subsection{Dependencias principales}

Tratamos de mantener las librerías al mínimo para no hacer tan pesado el contenedor:
\begin{itemize}
  \item \texttt{fastapi} y \texttt{uvicorn}: Para montar la API rapidísimo.
  \item \texttt{sentence-transformers}: Para sacar los embeddings de 384 dimensiones.
  \item \texttt{umap-learn} y \texttt{hdbscan}: Para la reducción dimensional y el clustering no supervisado.
  \item \texttt{spacy}: Para el filtrado NER y anonimización de datos sensibles.
  \item \texttt{chromadb}: Como base de datos vectorial in-memory.
\end{itemize}

\subsection{Estrategia de pruebas}

Nuestro principal objetivo de calidad fue garantizar que el servidor mantuviera su estabilidad incluso ante peticiones malformadas o inesperadas. Por ello, la estrategia de pruebas se centró fuertemente en la robustez de la API y el manejo de excepciones.

\begin{tabular}{@{}lp{9.5cm}@{}}
  \toprule
  Tipo & Qué valida \\
  \midrule
  Validadores & Confirma que el sistema rechace archivos sin "Objetivos" o "Alcance" (400). \\
  Integración & Asegura que el modelo procese el documento y Ollama retorne el JSON esperado. \\
  Contrato & Valida que el frontend siempre reciba los campos estructurados (200). \\
  Carga ligera & Pruebas manuales concurrentes para monitorear el consumo de memoria de Ollama. \\
  \bottomrule
\end{tabular}

\subsection{Colección Postman}

Para facilitar la verificación del sistema, estructuramos una colección completa en Postman. De esta manera, cualquier desarrollador o evaluador puede ejecutar las pruebas automatizadas y confirmar el correcto funcionamiento de los flujos.

\begin{itemize}
  \item Archivo en el repositorio: \texttt{collections/corvus\_clustering.postman\_collection.json}.
  \item Entorno configurado contra el servidor de producción: \texttt{http://207.180.215.71:3000}.
  \item Casos de uso validados:
    \begin{enumerate}
      \item \textbf{Pre-validación exitosa (200):} Envío de un documento estructurado, retornando el porcentaje de colisión.
      \item \textbf{Análisis profundo (200):} Consumo del endpoint de Ollama utilizando un proyecto real.
      \item \textbf{Historial paginado (200):} Verificación de que la ruta \texttt{/inference-history} retorne los datos correctamente paginados.
      \item \textbf{Payload incompleto (422):} Petición enviada sin el archivo adjunto requerido.
      \item \textbf{Falta sección obligatoria (400):} Envío de un texto válido pero omitiendo deliberadamente la sección "Alcance".
      \item \textbf{Recuperación de Draft (200):} Solicitud del último análisis rápido almacenado, evitando el reprocesamiento.
    \end{enumerate}
\end{itemize}

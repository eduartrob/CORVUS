\subsection{Base URL y versionado}

\begin{itemize}
  \item Base URL de producción: \texttt{http://207.180.215.71:3000} (Servidor Contabo)
  \item Prefijo de API: \texttt{/api/v1/clustering}
  \item OpenAPI: La documentación interactiva generada automáticamente por FastAPI se encuentra disponible en \texttt{/api/v1/clustering/docs}.
\end{itemize}

\subsection{Ejemplo de inferencia}

Dado que procesamos archivos físicos, las peticiones no utilizan JSON plano, sino el estándar \texttt{multipart/form-data}. A continuación, presento un ejemplo de cómo la aplicación móvil interactúa con el endpoint de validación:

\textbf{Request}

\begin{lstlisting}[style=jsonstyle]
POST /api/v1/clustering/pre-validate-proposal
Content-Type: multipart/form-data

user_id="eduart_001"
file=@"mi_propuesta.pdf"
\end{lstlisting}

\textbf{Response 200 (Análisis Rápido)}

\begin{lstlisting}[style=jsonstyle]
{
  "status": "success",
  "academic_alignment": 80,
  "collision_risk_pct": 12.5,
  "collision_risk_level": "Bajo",
  "positive_feedback": [
    "Originalidad alta. No se encontraron proyectos similares en el histórico."
  ],
  "areas_of_improvement": []
}
\end{lstlisting}

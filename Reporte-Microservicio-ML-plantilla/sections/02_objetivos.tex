\subsection{Objetivo general}

Nuestro objetivo en esta fase fue desarrollar un microservicio REST desde cero que se encargara de ejecutar un algoritmo de \textbf{aprendizaje no supervisado}. La meta principal era que este backend pudiera recibir los textos, identificar patrones inusuales o posibles colisiones entre proyectos, persistir los datos para el historial, y quedar completamente preparado para integrarse con nuestra aplicación móvil sin generar conflictos.

\subsection{Objetivos específicos}

Para lograr esto, estructuramos el trabajo de la siguiente manera:

\begin{enumerate}
  \item Construir un pipeline de preprocesamiento robusto utilizando NLP (SpaCy) para limpiar y anonimizar los PDFs antes de enviarlos al modelo.
  \item Entrenar nuestro pipeline de \textbf{HDBSCAN + UMAP} utilizando un corpus sintético, documentando toda la evidencia y métricas en un Jupyter Notebook.
  \item Exponer los endpoints mediante FastAPI garantizando que siempre retornen un JSON bien estructurado, asegurando consistencia en las respuestas.
  \item Integrar SQLite para almacenar un registro detallado de cada archivo procesado, incluyendo su porcentaje de colisión y nivel de riesgo.
  \item Generar la documentación interactiva con Swagger de forma automática y validar la estabilidad de la API mediante una colección de pruebas en Postman.
  \item Explicar mediante un video demostrativo cómo la aplicación móvil en Flutter consumirá esta API en la etapa final.
\end{enumerate}

\subsection{Criterios de aceptación mínimos}

Para validar que cumplimos con los requerimientos, definimos los siguientes puntos de control:

\begin{tabular}{@{}p{5.5cm}p{7.8cm}@{}}
  \toprule
  Requisito & Evidencia esperada \\
  \midrule
  Inferencia & Endpoint POST completamente operativo que retorne el JSON con el porcentaje de colisión. \\
  Historial & Endpoint GET con paginación implementada para no saturar la vista del administrador. \\
  Validación & FastAPI retornando errores HTTP 422 o 400 cuando se envían documentos inválidos o incompletos. \\
  Persistencia & Los registros almacenados correctamente en \texttt{inference\_log.db} sin excepciones. \\
  Documentación & La interfaz de Swagger (\texttt{/docs}) accesible y funcional en el servidor de producción. \\
  \bottomrule
\end{tabular}

PROBLEMÁTICA
La carencia de herramientas automatizadas para Sistemas / Bases de Datos genera cuellos de botella y errores frecuentes. Los usuarios dedican demasiado tiempo a procesos repetitivos debido a la falta de centralización y digitalización en sus actividades cotidianas.

OBJETIVO GENERAL
Desarrollar una solución integral para automatizar el flujo de Motor de Base de Datos Relacional SQL Embebido desde Cero (TinyDBEngine), mejorando significativamente los tiempos de respuesta y la precisión operativa de las tareas asociadas.

OBJETIVOS ESPECÍFICOS
- Diseñar la arquitectura del software contemplando los requerimientos de la categoría Sistemas / Bases de Datos.
- Programar e integrar las funcionalidades básicas como: Analizador de consultas (Parser) personalizado para comandos DDL y DML (CREATE TABLE, INSERT, SELECT, WHERE)., Estructura de almacenamiento en disco basada en archivos binarios con paginación fija de datos..
- Implementar controles de seguridad y almacenamiento local robustos.
- Validar la solución ante pruebas de carga y experiencia de usuario.

JUSTIFICACIÓN
Este proyecto se justifica por la necesidad de resolver problemáticas de Sistemas / Bases de Datos. Mediante la implementación de Motor de Base de Datos Relacional SQL Embebido desde Cero (TinyDBEngine), se pretende brindar una alternativa viable y moderna que reduzca los tiempos de trabajo manual y fomente la automatización.

USUARIOS FINALES
- **Administrador del Sistema:** Administra usuarios, configuraciones y emite reportes.
- **Usuario Operativo:** Interactúa con las pantallas operativas principales del sistema.

LISTA DE FUNCIONALIDADES POR TIPO DE USUARIO
**Funcionalidades de Administrador:**
1. Gestión de accesos y monitor de rendimiento.
2. Triage y auditorías.

**Funcionalidades de Usuario Operativo:**
1. Dashboard interactivo.
2. Acceso a las utilidades principales: Analizador de consultas (Parser) personalizado para comandos DDL y DML (CREATE TABLE, INSERT, SELECT, WHERE)., Estructura de almacenamiento en disco basada en archivos binarios con paginación fija de datos..

LISTA DE TECNOLOGÍAS A APLICAR

**Bases de Datos y Otros:** PostgreSQL / SQLite
**Frontend / Interfaz:** Desarrollo responsivo y dinámico.
